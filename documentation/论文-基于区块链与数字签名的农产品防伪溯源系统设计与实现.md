# 基于区块链与数字签名的农产品防伪溯源系统设计与实现

## 摘要

随着人们对食品安全的日益关注，农产品溯源系统成为保障食品安全的重要手段。然而，传统溯源系统存在数据易篡改、信息不透明、多方信任缺失等问题。本文设计并实现了一种基于区块链与数字签名的农产品防伪溯源系统，利用区块链技术的链式校验与防篡改特性，结合RSA数字签名算法确保数据的完整性和不可抵赖性。系统采用B/S架构，前端使用Vue.js框架，后端采用Spring Boot开发，业务数据使用MySQL存储并通过MyBatis-Plus访问，区块链核心使用Java实现轻量级单机原型，通过SHA-256哈希算法和Merkle树结构保证数据安全。系统实现了产品注册、溯源记录添加、溯源查询、区块链统计等核心功能，并生成二维码便于消费者查询。测试结果表明，系统在毕业设计原型验证场景下运行稳定，能够完成基本的防篡改校验与溯源展示，为农产品溯源提供了可验证的技术方案。

**关键词：** 区块链；数字签名；农产品溯源；防伪；SHA-256；RSA

---

## Abstract

With increasing attention to food safety, agricultural product traceability systems have become an important means of ensuring food security. However, traditional traceability systems suffer from issues such as data tampering, information opacity, and lack of multi-party trust. This paper designs and implements a blockchain and digital signature-based anti-counterfeiting traceability system for agricultural products. The system leverages blockchain-based chained verification and tamper-evident characteristics, combined with the RSA digital signature algorithm to ensure data integrity and non-repudiation. The system adopts a B/S architecture, uses Vue.js for the frontend, Spring Boot for the backend, MySQL with MyBatis-Plus for business data persistence, and a lightweight single-node blockchain prototype implemented in Java. SHA-256 hashing and a Merkle tree structure are used to support data verification. The system implements core functions including product registration, traceability record addition, traceability query, and blockchain statistics, and generates QR codes for consumer queries. Test results show that the system runs stably in a graduation-project prototype environment and can complete basic anti-tampering verification and traceability display.

**Keywords:** Blockchain; Digital Signature; Agricultural Product Traceability; Anti-counterfeiting; SHA-256; RSA

## 第一章 绪论

### 1.1 研究背景与意义

农产品质量安全关系到人民群众的身体健康和生命安全，是社会关注的热点问题。近年来，食品安全事件频发，如"三聚氰胺奶粉"、"瘦肉精"、"地沟油"等事件，严重损害了消费者的权益，也对农产品行业的健康发展造成了不良影响。根据国家统计局数据，2023年我国农产品总产量超过6.8亿吨，农产品流通规模超过15万亿元，如何保障如此庞大体量农产品的质量安全，成为亟待解决的重要课题。

传统的农产品溯源系统主要采用中心化数据库管理模式，存在以下突出问题：

（1）**数据易篡改**：中心化数据库的管理员可以任意修改数据，溯源信息的真实性无法得到有效保障。

（2）**信息不透明**：消费者难以获取完整的供应链信息，各环节数据相互隔离，形成"信息孤岛"。

（3）**多方信任缺失**：供应链各环节主体之间缺乏有效的信任机制，数据共享困难，责任追溯困难。

（4）**防伪能力不足**：传统防伪技术如防伪标签、二维码等容易被仿制，无法从根本上解决假冒伪劣问题。

区块链技术作为一种分布式账本技术，具有去中心化、不可篡改、可追溯等特点，为解决上述问题提供了新的技术路径。区块链通过密码学算法保证数据安全，通过链式结构和哈希校验增强数据可信度。将区块链技术应用于农产品溯源领域，可以有效提升溯源信息的可信度和透明度，构建多方参与的信任机制。

数字签名技术是保障数据完整性和不可抵赖性的重要手段。通过RSA非对称加密算法，每个参与方使用私钥对数据进行签名，其他方可以使用公钥验证签名的有效性，从而确保数据的真实性和来源的可靠性。

本研究将区块链技术与数字签名技术相结合，设计并实现了一套农产品防伪溯源系统，对于提升农产品质量安全水平、保护消费者权益、促进农产品行业健康发展具有重要的理论意义和实践价值。

## 第二章 相关技术概述

### 2.1 区块链技术

#### 2.1.1 区块链基本概念

区块链（Blockchain）是一种分布式数据库技术，其核心特征是将数据以区块的形式链接起来，形成一个不可篡改的链式结构。每个区块包含一批交易数据，并通过密码学哈希函数与前一个区块关联，从而保证数据的完整性和不可篡改性。

区块链技术最早由中本聪在2008年提出，作为比特币的底层技术。随着技术的发展，区块链已经从数字货币领域扩展到金融、供应链、医疗、政务等多个领域。

区块链的主要特点包括：

（1）**去中心化**：区块链网络中的节点地位平等，没有中心化的管理机构，数据由全网节点共同维护。

（2）**不可篡改**：一旦数据被写入区块链，就无法被修改或删除，保证了数据的历史可追溯性。

（3）**透明性**：区块链上的数据对所有节点可见，提高了信息的透明度。

（4）**可追溯性**：通过链式结构，可以追溯到数据的来源和历史变更记录。

（5）**智能合约**：可以在区块链上执行自动化的业务逻辑，实现可编程的信任。本文主要聚焦轻量级区块链存证与数字签名实现，未直接实现智能合约模块。

#### 2.1.2 区块链数据结构

区块链的基本数据结构包括区块头和区块体两部分。

**区块头**包含以下字段：

- **前一区块哈希（Previous Hash）**：指向前一个区块的哈希值，形成链式结构。
- **Merkle根（Merkle Root）**：所有交易的Merkle树根哈希，用于快速验证交易完整性。
- **时间戳（Timestamp）**：区块创建的时间。
- **难度值（Difficulty）**：挖矿难度目标。
- **随机数（Nonce）**：用于工作量证明的计数器。

**区块体**包含交易列表，每个交易记录了具体的业务数据。

#### 2.1.3 哈希函数与Merkle树

**哈希函数**是区块链技术的核心组件，用于将任意长度的数据映射为固定长度的哈希值。区块链中常用的哈希函数是SHA-256（Secure Hash Algorithm 256-bit），由美国国家安全局设计。SHA-256具有以下特性：

（1）**单向性**：从哈希值无法反推出原始数据。
（2）**抗碰撞性**：很难找到两个不同的数据产生相同的哈希值。
（3）**雪崩效应**：输入数据的微小变化会导致哈希值的巨大变化。

**Merkle树**是一种二叉树结构，用于高效地验证大量数据的完整性。在区块链中，所有交易的哈希值作为叶子节点，通过两两配对计算父节点的哈希值，最终形成一个根哈希值（Merkle根）。Merkle树的优势在于可以高效地验证单个交易是否包含在区块中，而无需下载所有交易。

#### 2.1.4 工作量证明机制

工作量证明（Proof of Work，PoW）是区块链中常用的共识机制，通过计算难题来竞争记账权。在PoW机制中，矿工需要找到一个随机数（Nonce），使得区块头的哈希值满足特定的难度要求（如哈希值以若干个0开头）。

PoW机制的特点：

（1）**计算困难**：找到满足条件的Nonce需要大量的计算尝试。
（2）**验证容易**：一旦找到Nonce，验证其正确性非常简单。
（3）**安全性高**：攻击者需要控制全网大部分算力才能篡改数据。

本系统采用简化的PoW机制，通过调整难度值来控制挖矿速度。

### 2.2 数字签名技术

#### 2.2.1 数字签名概述

数字签名是一种用于验证数字信息真实性和完整性的密码学技术。数字签名可以确保：

（1）**身份认证**：确认消息确实来自声称的发送者。
（2）**数据完整性**：确认消息在传输过程中未被篡改。
（3）**不可抵赖性**：发送者无法否认自己发送过该消息。

#### 2.2.2 RSA算法原理

RSA是一种非对称加密算法，由Rivest、Shamir和Adleman在1977年提出。RSA算法基于大整数分解的数学难题，其安全性依赖于将两个大素数相乘容易，但对其乘积进行因数分解极其困难。

RSA算法包括密钥生成、加密和解密三个过程：

**密钥生成**：
（1）选择两个大素数p和q。
（2）计算n = p × q，n称为模数。
（3）计算欧拉函数φ(n) = (p-1) × (q-1)。
（4）选择公钥指数e，满足1 < e < φ(n)且gcd(e, φ(n)) = 1。
（5）计算私钥指数d，满足d × e ≡ 1 (mod φ(n))。
（6）公钥为(n, e)，私钥为(n, d)。

**加密过程**：c = m^e mod n，其中m为明文，c为密文。

**解密过程**：m = c^d mod n。

#### 2.2.3 数字签名实现

基于RSA的数字签名过程如下：

**签名过程**：
（1）计算消息的哈希值：h = H(m)，其中H为哈希函数（如SHA-256）。
（2）使用私钥对哈希值签名：s = h^d mod n。
（3）将签名s与消息m一起发送。

**验证过程**：
（1）计算接收消息的哈希值：h' = H(m)。
（2）使用公钥验证签名：h = s^e mod n。
（3）比较h和h'，如果相等则签名有效。

在本系统中，每个用户生成一对RSA密钥，私钥用于对溯源数据进行签名，公钥用于验证签名的有效性。区块链上存储交易数据和数字签名，任何人都可以使用公钥验证数据的真实性。

## 第三章 系统需求分析

### 3.1 业务需求分析

#### 3.1.1 农产品溯源业务流程

农产品从生产到消费的全生命周期包括以下环节：

（1）**生产环节**：种植或养殖过程，包括播种、施肥、灌溉、病虫害防治、收获等。

（2）**加工环节**：对农产品进行清洗、分拣、包装、深加工等处理。

（3）**仓储环节**：农产品在仓库中的储存管理，包括入库、库存管理、出库等。

（4）**运输环节**：农产品从产地到销地的物流运输过程。

**区块哈希计算**：
```java
public String calculateHash() {
    try {
        String data = index + previousHash + merkleRoot + timestamp + nonce + difficulty;
        MessageDigest digest = MessageDigest.getInstance("SHA-256");
        byte[] hash = digest.digest(data.getBytes("UTF-8"));
        
        StringBuilder hexString = new StringBuilder();
        for (byte b : hash) {
            String hex = Integer.toHexString(0xff & b);
            if (hex.length() == 1) hexString.append('0');
            hexString.append(hex);
        }
        return hexString.toString();
    } catch (Exception e) {
        throw new RuntimeException("计算哈希失败", e);
    }
}
```

**Merkle根计算**：
```java
public String calculateMerkleRoot() {
    if (transactions == null || transactions.isEmpty()) {
        return "";
    }
    
    List<String> treeLayer = new ArrayList<>();
    for (Transaction tx : transactions) {
        treeLayer.add(tx.getTransactionHash());
    }
    
    while (treeLayer.size() > 1) {
        List<String> newLayer = new ArrayList<>();
        for (int i = 0; i < treeLayer.size(); i += 2) {
            String left = treeLayer.get(i);
            String right = (i + 1 < treeLayer.size()) ? treeLayer.get(i + 1) : left;
            newLayer.add(applySha256(left + right));
        }
        treeLayer = newLayer;
    }
    
    return treeLayer.get(0);
}
```

#### 5.2.2 数字签名实现

数字签名使用Java的Security库实现RSA签名和验证。

**密钥生成**：
```java
public static Map<String, String> generateKeyPair() {
    try {
        KeyPairGenerator keyGen = KeyPairGenerator.getInstance("RSA");
        keyGen.initialize(2048);
        KeyPair keyPair = keyGen.generateKeyPair();
        
        PublicKey publicKey = keyPair.getPublic();
        PrivateKey privateKey = keyPair.getPrivate();
        
        String publicKeyBase64 = Base64.getEncoder().encodeToString(publicKey.getEncoded());
        String privateKeyBase64 = Base64.getEncoder().encodeToString(privateKey.getEncoded());
        
        Map<String, String> keys = new HashMap<>();
        keys.put("publicKey", publicKeyBase64);
        keys.put("privateKey", privateKeyBase64);
        
        return keys;
    } catch (Exception e) {
        throw new RuntimeException("生成密钥对失败", e);
    }
}
```

**签名过程**：
```java
public String sign(String data, String privateKeyBase64) {
    try {
        byte[] privateKeyBytes = Base64.getDecoder().decode(privateKeyBase64);
        PKCS8EncodedKeySpec keySpec = new PKCS8EncodedKeySpec(privateKeyBytes);
        KeyFactory keyFactory = KeyFactory.getInstance("RSA");
        PrivateKey privateKey = keyFactory.generatePrivate(keySpec);
        
        Signature signature = Signature.getInstance("SHA256withRSA");
        signature.initSign(privateKey);
        signature.update(data.getBytes(StandardCharsets.UTF_8));
        
        byte[] signatureBytes = signature.sign();
        return Base64.getEncoder().encodeToString(signatureBytes);
    } catch (Exception e) {
        throw new RuntimeException("数字签名失败", e);
    }
}
```

**验证过程**：
```java
public boolean verify(String data, String signatureBase64, String publicKeyBase64) {
    try {
        byte[] publicKeyBytes = Base64.getDecoder().decode(publicKeyBase64);
        X509EncodedKeySpec keySpec = new X509EncodedKeySpec(publicKeyBytes);
        KeyFactory keyFactory = KeyFactory.getInstance("RSA");
        PublicKey publicKey = keyFactory.generatePublic(keySpec);
        
        byte[] signatureBytes = Base64.getDecoder().decode(signatureBase64);
        
        Signature signature = Signature.getInstance("SHA256withRSA");
        signature.initVerify(publicKey);
        signature.update(data.getBytes(StandardCharsets.UTF_8));
        
        return signature.verify(signatureBytes);
    } catch (Exception e) {
        return false;
    }
}
```

                                String producerId, String origin, String description) {

    TraceRecord record = TraceRecord.create(productId, "PRODUCE", producerId,
            producer.getRealName(), "PRODUCER", origin, "产品生产注册");
    
    Transaction transaction = createTransaction(product, record, producer);
    String signature = digitalSignature.signTransaction(transaction, producer.getPrivateKey());
    transaction.setDigitalSignature(signature);
    
    // 添加到区块链
    blockchain.addTransaction(transaction);
    Block newBlock = blockchain.minePendingTransactions(producerId);
    
    // 生成二维码并保存
    String qrCode = generateQRCode(productId);
    product.setQrCode(qrCode);
    productRepository.save(product);
    
    return product;

## 第六章 总结与展望

### 6.1 研究工作总结

本文设计并实现了一套基于区块链与数字签名的农产品防伪溯源系统，主要完成了以下工作：

（1）**需求分析**：深入分析了农产品溯源的业务需求和系统功能需求，明确了系统的设计目标。

（2）**技术研究**：研究了区块链技术和数字签名技术的基本原理，包括区块链的数据结构、共识机制，以及RSA数字签名算法。

（3）**系统设计**：设计了系统的总体架构、功能模块、数据库结构和区块链数据结构，采用B/S架构，前后端分离设计。

（4）**系统实现**：使用SpringBoot开发后端服务，使用Vue.js开发前端界面，使用Java实现轻量级区块链核心功能，包括区块创建、Merkle树计算、工作量证明挖矿等。

（5）**数字签名**：基于RSA算法实现了数字签名功能，为每条溯源记录生成数字签名，确保数据的完整性和不可抵赖性。

（6）**系统测试**：对系统进行了功能测试、性能测试和安全测试，验证了系统的正确性、可靠性和安全性。

本系统的主要特点包括：

（1）**数据校验能力较强**：采用区块链结构和数字签名对溯源数据进行校验，在系统运行周期内能够有效发现异常改动，提高数据可信度。

（2）**身份可验证**：使用数字签名技术，每条记录都有明确的签名者，任何人都可以验证记录的真实性和来源。

（3）**全程可追溯**：记录农产品从生产到销售的全生命周期信息，消费者可以查询完整的溯源历史。

（4）**防伪能力强**：结合二维码技术和区块链验证，有效防止假冒伪劣产品。

（5）**用户体验好**：提供简洁直观的Web界面，支持二维码扫描查询，方便消费者使用。

### 6.2 存在的不足

本系统在研究和实现过程中还存在一些不足之处：

（1）**区块链规模**：本系统采用轻量级区块链实现，适用于中小规模应用场景，对于大规模商业应用可能需要考虑联盟链方案。

（2）**共识机制**：当前采用简化的PoW共识机制，挖矿过程消耗计算资源，可以考虑采用更高效的共识算法。

（3）**链数据持久化**：当前轻量级区块链以内存方式运行，服务重启后链状态无法完整保留，后续需要补充持久化方案。

（4）**私钥保护**：当前系统已对密码进行哈希存储，但私钥保护仍需进一步增强，如增加加密存储或密钥托管方案。

（5）**移动端支持**：当前系统主要面向Web端，移动端应用的支持还不够完善。

（6）**物联网集成**：系统预留了物联网数据接口，但实际集成还需要进一步完善。

### 6.3 未来展望

针对存在的不足，未来可以从以下方向进行改进和扩展：

（1）**联盟链升级**：将轻量级区块链升级为联盟链（如Hyperledger Fabric或FISCO BCOS），支持多节点共识和企业级应用。

（2）**共识优化**：研究和实现更高效的共识算法，如PBFT（实用拜占庭容错）或PoS（权益证明），降低能源消耗。

（3）**存储优化**：采用IPFS等分布式存储技术，优化大数据量的存储和访问性能。

（4）**移动应用**：开发配套的移动应用程序，支持扫码查询、拍照上传等功能。

（5）**物联网集成**：集成温度传感器、湿度传感器等物联网设备，实现环境数据的自动采集和上链。

（6）**规则自动化**：结合规则引擎或联盟链平台，进一步实现质量预警、流程校验等自动化业务逻辑。

（7）**跨链互操作**：研究与其他区块链系统的互操作方案，实现跨平台的溯源信息共享。

（8）**隐私保护**：研究零知识证明等隐私保护技术，在保障溯源透明度的同时保护商业隐私。

### 6.4 结语

农产品质量安全是关系到国计民生的重要问题，区块链技术的出现为解决农产品溯源问题提供了新的技术路径。本系统将区块链技术与数字签名技术相结合，设计并实现了一套农产品防伪溯源系统，为农产品溯源提供了可靠的技术方案。

随着区块链技术的不断发展和完善，相信未来会有更多的农产品溯源系统采用区块链技术，为消费者提供更加安全、透明、可信的农产品溯源服务，为保障食品安全做出更大的贡献。
