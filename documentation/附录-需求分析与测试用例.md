## 2. 需求分析

### 2.1 业务目标

系统面向农产品生产、加工和消费者查询场景，目标是在答辩演示条件下形成一个完整、可运行、可部署的溯源原型。

### 2.2 角色定义

- 生产者：注册产品，维护源头记录
- 加工者：补充加工环节记录
- 消费者：通过产品 ID 或二维码进行公开查询
- 管理展示用户：查看统计数据、区块信息和系统状态

### 2.3 功能需求

#### R1 用户注册

- 系统应支持用户注册
- 注册时应记录用户名、密码、用户类型、公司名称等信息
- 注册后应生成公私钥对

#### R2 用户登录

- 系统应支持用户名密码登录
- 登录成功后返回 JWT
- 非公开接口需要鉴权

#### R3 产品注册

- 系统应支持生产者注册产品
- 产品应包含名称、类别、产地、描述等字段
- 注册成功后应生成产品 ID 和二维码

#### R4 溯源记录新增

- 系统应支持对指定产品新增溯源记录
- 记录应包含操作类型、地点、详情等字段
- 记录新增时应生成签名交易并参与链上校验

#### R5 产品与溯源查询

- 系统应支持按产品 ID 查询
- 系统应支持二维码公开查询
- 查询结果应展示产品信息、溯源记录和校验状态

#### R6 区块链信息展示

- 系统应支持查看区块链统计信息
- 系统应支持区块列表展示
- 系统应支持交易或区块验证结果展示

#### R7 统计分析

- 系统应支持统计产品总数、用户总数、区块总数和记录总数
- 系统应支持按产品类别、产品状态、用户类型和操作类型进行统计展示

### 2.4 非功能需求

#### N1 可部署性

- 系统应支持本地运行
- 系统应支持 Docker Compose 一键部署

#### N2 可演示性

- 系统应支持桌面浏览器访问
- 系统应支持手机浏览器访问
- 系统应支持扫码公开查询

#### N3 安全性

- 密码不应明文存储
- 私钥不应直接明文对外返回
- 关键交易应经过签名校验

#### N4 可维护性

- 前后端应分层清晰
- 代码应支持 Git 版本管理
- 核心模块应有基本测试支撑

## 4. 核心测试用例

| 编号 | 测试目标 | 前置条件 | 操作步骤 | 预期结果 |
|---|---|---|---|---|
| TC-01 | 用户注册 | 系统正常运行 | 提交注册信息 | 返回成功，密码不明文返回 |
| TC-02 | 用户登录 | 用户已注册 | 提交用户名密码 | 返回 JWT 与安全用户视图 |
| TC-03 | 非公开接口鉴权 | 未登录 | 访问受保护接口 | 被拒绝或提示未授权 |
| TC-04 | 产品注册 | 已登录生产者 | 提交产品信息 | 返回产品 ID 和产品详情 |
| TC-05 | 产品列表查询 | 至少有一条产品 | 调用产品列表接口 | 返回产品列表 |
| TC-06 | 溯源记录新增 | 已登录且存在产品 | 提交溯源记录 | 返回新增成功，记录可查询 |
| TC-07 | 公开查询 | 产品存在 | 通过产品 ID 或二维码查询 | 展示产品信息和溯源记录 |
| TC-08 | 数据一致性校验 | 产品有链上记录 | 查询公开溯源结果 | 返回 `dataConsistent` |
| TC-09 | 区块链统计 | 系统正常运行 | 查询区块链统计接口 | 返回区块总数、交易总数、链状态 |
| TC-10 | 链状态持久化 | 已产生新区块 | 重启系统后重新查询 | 链状态可恢复 |
| TC-11 | 私钥加密存储 | 用户已注册 | 查看数据库私钥字段 | 不应为明文原值 |
| TC-12 | 前端构建 | 安装依赖完成 | 执行 `npm run build` | Web 前端构建通过 |
| TC-13 | 移动端 H5 构建 | 安装依赖完成 | 执行 `npm run build:h5` | H5 构建通过 |
