# 农产品防伪溯源系统 - 项目复现文档

## 1. 文档目的

本文档用于说明当前最终版本项目的本地运行方式、Docker 部署方式、基础验证步骤和常见问题排查方法。

说明边界：

- 本文档基于当前仓库最终实现编写
- 运行方案以 `MySQL + Spring Boot + Vue + uni-app H5` 为准
- 不再使用旧版本中的 `H2`、`JPA` 或 `h2-console`

## 2. 运行环境

### 2.1 建议版本

- JDK 11
- Maven 3.8 及以上
- Node.js 16 及以上
- npm 8 及以上
- MySQL 8
- Docker 与 Docker Compose

### 2.2 项目结构

```text
agri-traceability-system/
├─ backend/                 后端服务
├─ frontend/                Web 前端
├─ mobile/                  uni-app H5 移动端
├─ documentation/           项目文档
├─ presentation/            答辩材料
├─ docker-compose.yml       一键部署配置
└─ .env.example             环境变量示例
```

## 3. 本地运行

### 3.1 启动后端

先准备本地 MySQL，并设置环境变量：

```powershell
$env:DB_URL="jdbc:mysql://127.0.0.1:3306/agritrace?createDatabaseIfNotExist=true&useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai"
$env:DB_USERNAME="root"
$env:DB_PASSWORD="你的数据库密码"
$env:JWT_SECRET="一段足够长的随机字符串"
$env:APP_KEY_SECRET="另一段足够长的随机字符串"
```

启动命令：

```bash
cd backend
mvn spring-boot:run
```

默认接口地址：

- `http://localhost:8080/api/health`
- `http://localhost:8080/api/products`

### 3.2 启动 Web 前端

```bash
cd frontend
npm install
npm run serve
```

默认访问地址：

- `http://localhost:8080` 为后端接口
- `http://localhost:3000` 为开发模式前端页面

### 3.3 启动移动端 H5

```bash
cd mobile
npm install
npm run dev:h5
```

默认情况下移动端读取部署域名；如果做本地联调，可在本地存储中覆盖 `apiBaseUrl`。

## 4. Docker 部署

### 4.1 准备环境变量

复制示例文件：

```bash
cp .env.example .env
```

至少配置以下变量：

- `MYSQL_ROOT_PASSWORD`
- `JWT_SECRET`
- `APP_KEY_SECRET`

### 4.2 启动服务

```bash
docker compose up -d --build
```

默认暴露端口：

- 前端：`8088`
- 后端：`8080`

### 4.3 容器说明

- `mysql`：业务数据库
- `backend`：Spring Boot 服务
- `frontend`：Nginx 承载的 Web 页面

链状态通过 Docker 卷和 `BLOCKCHAIN_STATE_FILE` 持久化保存。

## 6. 核心验证步骤

### 6.1 健康检查

```bash
curl http://localhost:8080/api/health
```

预期结果：

- `status` 为 `UP`
- `blockchainValid` 为 `true`

### 6.2 用户注册与登录

注册：

```bash
curl -X POST http://localhost:8080/api/users/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser",
    "password": "123456",
    "realName": "Test User",
    "userType": "PRODUCER",
    "companyName": "Test Farm"
  }'
```

登录：

```bash
curl -X POST http://localhost:8080/api/users/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser",
    "password": "123456"
  }'
```

预期结果：

- 登录成功
- 返回 JWT
- 响应中不应泄露密码和私钥

### 6.3 注册产品

```bash
curl -X POST http://localhost:8080/api/products/register \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{
    "productName": "Demo Tomato",
    "productCategory": "FR",
    "origin": "Shouguang",
    "description": "Graduation project demo product"
  }'
```

预期结果：

- 返回产品 ID
- 产品可在 `/api/products` 中看到
- 页面可生成公开查询二维码

### 6.4 添加溯源记录

```bash
curl -X POST http://localhost:8080/api/trace/add \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{
    "productId": "<productId>",
    "operationType": "PROCESS",
    "location": "Jinan Center",
    "operationDetail": "Washed and packed for demo"
  }'
```

预期结果：

- 新增记录成功
- 交易进入链上校验流程
- 区块统计数据发生变化

### 6.5 公开查询

```bash
curl http://localhost:8080/api/trace/<productId>
```

预期结果：

- 返回产品信息与溯源记录
- 返回 `dataConsistent`
- 返回 `blockchainValid`

## 9. 常见问题排查

### 9.1 端口被占用

如果 `8080` 或 `8088` 被占用，可以：

- 修改本地启动端口
- 或关闭占用端口的现有进程

### 9.2 MySQL 连接失败

重点检查：

- `DB_URL`
- `DB_USERNAME`
- `DB_PASSWORD`
- MySQL 服务是否已启动

### 9.3 前端请求失败

重点检查：

- 开发代理是否指向正确的后端地址
- JWT 是否已写入请求头
- 浏览器是否缓存了旧前端静态资源

### 9.4 构建成功但页面显示旧内容

建议执行：

- 浏览器 `Ctrl + F5` 强制刷新
- 重新部署前端容器

### 9.5 链状态异常

重点检查：

- `BLOCKCHAIN_STATE_FILE` 是否正确配置
- Docker 卷是否被清理
- 服务器是否误删链状态文件
